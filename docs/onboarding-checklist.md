# Onboarding checklist

Copy this checklist into your ticket if you need a single place to track progress.

## Intake
- [ ] Agency, business owner, technical owner, and requestor identified
- [ ] Proposed org name and purpose statement drafted
- [ ] Two proposed org admins identified
- [ ] Users estimated for launch, 12 months, and 36 months
- [ ] Paid features identified
- [ ] Existing Azure subscription status identified
- [ ] Need for new or retroactive Azure subscription creation noted, if applicable
- [ ] Outside collaborators need identified
- [ ] SSO dependency identified
- [ ] Data sensitivity and cloud/cyber concerns identified
- [ ] `[DECISION NEEDED]` Azure tenant membership question raised if it affects the request

## Funding approval
- [ ] Azure pricing calculator estimate created
- [ ] Estimate assumptions documented
- [ ] Higher non-enterprise estimate accepted rather than discounted
- [ ] Payment option selected: PayGo or Monetary Commit
- [ ] For PayGo: OMB approval for increased spending attached or routed
- [ ] For Monetary Commit: formal quote attached
- [ ] For Monetary Commit: estimate based on 36 months of expected usage
- [ ] For Monetary Commit: period of performance ending September 30, 2030 noted
- [ ] For Monetary Commit: use-it-or-lose-it nature acknowledged
- [ ] Billing owner identified
- [ ] Users with Copilot seats in other orgs flagged, if applicable
- [ ] Funding decision recorded

## Cloud/cyber approval
- [ ] Routine review path confirmed or escalation reason documented
- [ ] Service summary completed
- [ ] Scope of use documented
- [ ] Data classification documented
- [ ] Identity and SSO requirements documented
- [ ] Baseline controls documented
- [ ] Cloud integrations documented
- [ ] AI, Copilot, and MCP usage documented, if applicable
- [ ] Approval decision recorded

## Organization configuration
- [ ] Org name approved
- [ ] Org purpose approved
- [ ] Two org admins approved
- [ ] Team and membership model documented
- [ ] Outside collaborator rules documented
- [ ] Repository defaults and baseline controls documented
- [ ] Paid features at launch documented
- [ ] Copilot model, agent, and MCP choices documented, if applicable
- [ ] Org admin operating responsibilities acknowledged
- [ ] Dormant user management process documented
- [ ] Successor path documented for org admins leaving the city

## Implementation and closeout
- [ ] Funding approval routed to ODA
- [ ] Azure subscription connection need identified
- [ ] Azure admin handoff completed if subscription connection is needed
- [ ] Organization created or updated
- [ ] Approved org admins added
- [ ] Access tested
- [ ] Major setup decisions recorded in the ticket
- [ ] Request closed or moved into steady-state support
