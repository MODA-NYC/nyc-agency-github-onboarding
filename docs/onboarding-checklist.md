# Onboarding checklist

Copy this checklist into your ticket if you need a single place to track progress.

## Intake
- [ ] Agency, business owner, technical owner, and requestor identified
- [ ] Purpose statement drafted
- [ ] Two proposed org admins identified
- [ ] Users and paid features estimated for the chosen funding path
- [ ] Visual Studio license count check completed (users with Visual Studio licenses tied to the same email do not need an additional GitHub Enterprise license)
- [ ] Paid features identified
- [ ] Existing Azure subscription status checked
- [ ] ODA / Cloud Services asked to identify Azure subscription and point of contact, if needed
- [ ] Migration of existing GitHub org or repositories assessed
- [ ] Outside collaborators need identified
- [ ] SSO required; plan and timeline captured `[DECISION NEEDED: set required SSO integration deadline to 60 or 90 days after onboarding]`
- [ ] Data sensitivity and cloud/cyber concerns identified
- [ ] `[DECISION NEEDED]` Azure tenant membership question raised if it affects the request

## ServiceNow tracking
- [ ] Parent ServiceNow ticket created
- [ ] Workstream tracker in parent ticket maintained (funding, cloud/cyber, org configuration)

## Funding approval
- [ ] Azure subscription status confirmed
- [ ] ODA / Cloud Services asked to identify subscription and Azure point of contact, if needed
- [ ] Visual Studio license count check completed
- [ ] Azure pricing calculator estimate created
- [ ] Estimate assumptions documented
- [ ] Higher non-enterprise estimate accepted rather than discounted
- [ ] Payment option selected: Pay-As-You-Go or Monetary Commit
- [ ] For Pay-As-You-Go: OMB approval for increased spending attached or routed
- [ ] For Monetary Commit: formal quote from Microsoft attached
- [ ] For Monetary Commit: estimate based on 3 years of expected usage
- [ ] For Monetary Commit: period of performance ending September 30, 2030 noted
- [ ] For Monetary Commit: use-it-or-lose-it nature acknowledged
- [ ] Billing owner identified
- [ ] Users with Copilot seats in other orgs flagged, if applicable
- [ ] Funding decision recorded

## Cloud/cyber approval
- [ ] Routine review path confirmed or escalation reason documented
- [ ] Standard boilerplate included in review request
- [ ] Agency-specific details completed
- [ ] Data classification documented
- [ ] SSO plan and timeline documented
- [ ] Existing org / repo migration documented, if applicable
- [ ] AI, Copilot, and MCP usage documented, if applicable
- [ ] Cloud/cyber review ticket number linked from parent ticket, if applicable
- [ ] Approval decision recorded

## Organization configuration
- [ ] Org purpose documented (ODA assigns org name)
- [ ] Two org admins approved
- [ ] Team and membership model documented
- [ ] Outside collaborator rules documented
- [ ] Repository defaults and baseline controls documented
- [ ] Migration plan documented, if applicable
- [ ] SSO integration plan and timeline documented
- [ ] Paid features at launch documented
- [ ] Copilot model, agent, and MCP choices documented, if applicable
- [ ] Org admin operating responsibilities acknowledged (see [Organization configuration](02-org-configuration.md))
- [ ] Dormant user management process documented
- [ ] Successor path documented for org admins leaving the city

## Implementation and closeout
- [ ] Funding approval routed to ODA
- [ ] Azure subscription connection coordinated by ODA
- [ ] Organization created or updated
- [ ] Migration of existing org / repos completed, if applicable
- [ ] Approved org admins added
- [ ] Access tested
- [ ] Major setup decisions recorded in the ticket
- [ ] Request closed or moved into steady-state support
